Class, Id, and Efficient CSS3

Classes and ID's are fundamental in understanding organization in HTML and CSS. Here is a brief blog outlining the proper uses of the two and more importantly, why they are proper.

Class

A class, like an ID is attribute a developer can set on an HTML tag. It can help organize an HTML page into sections which in turn makes it easier to style in CSS. Any given class name can be used an infinite amount of times in an HTML document.

Some developers don't know the full extent to which they can use classes. More than one can be used on the same element.

This demonstrates a 'div' using two classes on the same 'div'. If used properly, this technique can save you typing time, optimizing your CSS stylesheet.

ID's

The infamous ID is easy to use improperly. A given ID name can only be used once on an HTML doc. If an ID name is used more than once, 2 things happen: Your page fails validation. And it can cause any Javascript to act buggy. CSS can select specific ID's using the hash sign (#).

How to Use Them Efficiently

Keep in mind that every developer has a different style. That said, the way I typically organize my work flow is sticking Class attributes on tags I want to style, and ID's on elements that I want to call on with Javascript. Javascript relies heavily on ID's e.g. 'getElementById() function'. One could also make the argument that if someone has to review your code, it is easier if there's ID's are tied to javascript.

This subject seems to be one of personal choice but this is what works for me. Thanks for reading.
