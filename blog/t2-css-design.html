Absolutely Not Your Relative

A self-needed explanation on the differences between
Absolute, Relative, Fixed, & Static positioning in CSS.

February 13th 2015

The reason I'm writing about positioning in CSS is because I want to actually try and understand it myself. I've only been really exposed to learning CSS in the last week, and I'm really excited with the amazing things it's capable of.

So let's start with the default attribute for every element on the page. That is 'Static'. I just found out 5 minutes ago that the only reason you would ever want to use 'Static' is if you have to forcefully remove some positioning that was applied to an element that is out of your control. Gonna go delete a few lines of code now. Just kidding, I've never used 'Static' before, but I guess all it means is that the element you are working on will flow into the page as it normally would.

Now for 'Relative' positioning, the most mysterious to me. From what I've read it looks like 'Relative' positioning is basically the same as 'Static' positioning if you don't also apply some positioning attributes to the element. Like "top: 4px; left: 8px" etc. When you DO apply those attributes of positioning, the element will position itself relative to where it would have been if it were still a 'Static' element. At the same time, by making an element 'Relative' you are allowing it to use a z-index on that element which will enables you to decide which elements are in front of or behind other elements visually. Finally, by making an element have 'Relative' positioning you are also limiting the scope of any child element with 'Absolute' positioning enabled. We will get into 'Absolute' positioning next.

'Absolute' positioning allows the placement of an element anywhere on the page and it's positioning is set by the 'top', 'bottom', 'left', and 'right' attributes. The element that is set to 'Absolute' positioning is relative to the html element itself, meaning the entire page, or if it is nested within another element and that element has 'Relative' or 'Absolute' positioning it will be positioned relative to that element. Elements positioned using the 'Absolute' value will not be affected by, or affect, other elements on the page. This can be useful or pose problems depending on how it's implemented.

Finally, 'Fixed' positioning, when used on an element, makes that element's position relative to the browser window only. So no matter how long or wide the page is the element will stay in one position as all other elements move around behind it. This one is harder for me to put into words, but if you've seen a navigation bar that stays put on the side even as you scroll down the whole page then there is a possibility that 'Fixed' positioning was used.
